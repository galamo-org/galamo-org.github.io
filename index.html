About Galamo

Galamo is a modular, open-source Python toolkit for machine-learning-assisted galaxy analysis. Classifying galaxies by morphology and nuclear activity is key to understanding their evolution, but manual classification is infeasible for large surveys like SDSS and HST. Galamo addresses this by providing automated tools for morphological and spectroscopic classification, along with image enhancement capabilities, making reproducible galaxy science accessible for both education and research.

Core Modules

  • galaxy_morph: Performs supervised morphological classification using a Convolutional Neural Network (CNN) trained on the Galaxy Zoo dataset. It classifies galaxies into 36 detailed categories or simplified classes: *spiral*, *elliptical*, and *merger*.
  • bpt: Automates the classification of galaxies into AGN, composite, or star-forming based on the BPT diagram. It computes emission line ratios and compares them against the established boundaries from Kauffmann et al. (2003) and Kewley et al. (2001):
    Kauffmann et al. (2003): $$ \log_{10} \left( \frac{[\text{OIII}]}{\text{H}\beta} \right) = \frac{0.61}{\log_{10} \left( \frac{[\text{NII}]}{\text{H}\alpha} \right) - 0.05} + 1.3 $$ Kewley et al. (2001): $$ \log_{10} \left( \frac{[\text{OIII}]}{\text{H}\beta} \right) = \frac{0.61}{\log_{10} \left( \frac{[\text{NII}]}{\text{H}\alpha} \right) - 0.47} + 1.19 $$
  • psr: Enhances the quality and clarity of low-light astronomical images using a computer vision pipeline (denoising, contrast enhancement, gamma correction), serving as an effective preprocessing step before classification.

Dependencies & Availability

The package is written in Python 3.8+ and depends on Astropy, NumPy, Matplotlib, TensorFlow, Astroquery, and Scikit-learn. Galamo is publicly available on GitHub and PyPI.

Contributing to Galamo

Thank you for your interest in contributing to Galamo. This document outlines the guidelines and procedures for contributing. We welcome contributions that improve the software, documentation, or overall usability of the package.

Types of Contributions

We welcome the following types of contributions:

  • Reporting bugs
  • Proposing and implementing new features
  • Improving documentation and tutorials
  • Writing unit tests or enhancing test coverage
  • Refactoring or improving existing code

Bug Reports

To report a bug, please open an issue on the GitHub Issues page with the following information:

  • A clear and descriptive title
  • Steps to reproduce the issue
  • Expected and actual behavior
  • Environment details (operating system, Python version)
  • Any relevant error messages or tracebacks

Development Environment Setup

To contribute code, please follow these steps:

  1. Fork the repository on GitHub.
  2. Clone your forked repository: 
    git clone https://github.com/your-username/galamo.git
cd galamo
  3. Create and activate a virtual environment: 
    python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
  4. Install the package in editable mode with development dependencies: 
    pip install -e .[dev]

Pull Request Procedure

  1. Create a new branch for your work: 
    git checkout -b feature/my-new-feature
  2. Make your changes and commit them with a clear message: 
    git commit -m "feat: Add new feature"
  3. Push the changes to your fork: 
    git push origin feature/my-new-feature
  4. Open a pull request (PR) to the `main` branch of the `galamo-org/galamo` repository.
  5. In the PR description, explain your changes and link to any relevant issues.

Code of Conduct

All contributors are expected to adhere to the project's Code of Conduct. We are committed to providing a welcoming and inclusive environment for everyone. We appreciate your efforts to improve Galamo!
